## 附图占位格式

每个需要配图的位置必须输出附图建议，供 UX 设计师参考：

```markdown
> <!-- 附图占位：建议附上一张[图的类型]，[展示内容描述]，[图要传达的设计结论] -->
```

**图的类型**：示例图 / 对比图（推荐 vs. 不推荐）/ 场景图 / 状态图

**格式示例：**

```markdown
> <!-- 附图占位：建议附上一张示例图，展示主按钮（实心填充）、默认按钮（描边）、文本按钮三种形态并排，体现视觉层级差异 -->

> <!-- 附图占位：建议附上一张对比图，左侧展示同一页面只有一个主按钮（符合规范），右侧展示两个同等强调的主按钮并列（违反规范） -->

> <!-- 附图占位：建议附上一张场景图，展示表单底部「取消 → 重置 → 提交（主按钮）」的排列顺序，体现操作从轻到重的流转逻辑 -->
```

建议仅作参考，设计师按实际业务样式自行出图，不强制执行。
