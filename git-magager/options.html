Git Magager

Browser-native repository cloning

How it works

Click Clone on a GitHub or GitLab repository page. Chrome will ask where to save the repository, then Git Magager clones it directly into that folder.

The Git engine runs inside the extension. You do not need to run node server.js or keep a terminal open.

The initial browser clone downloads the latest commit and includes a working .git directory. Run git fetch --unshallow later if you need the complete history.

Compatibility

Direct folder access uses the browser's File System Access API. HTTPS repository URLs are supported; SSH-style URLs are automatically converted to HTTPS.
